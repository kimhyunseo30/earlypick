# EarlyPick

온라인 관심 신호와 시장 데이터를 비교 분석하여 식품 트렌드 후보를 조기에 탐색하고, 진입 타이밍 판단을 돕는 데이터 기반 의사결정 서비스입니다.

## 프로젝트 배경

식품 트렌드는 이미 유행한 이후 발견되는 경우가 많아, 상품 기획 및 마케팅 의사결정 시점이 늦어지는 문제가 있습니다.

EarlyPick은 네이버 쇼핑 인사이트 순위 데이터와 검색·쇼핑 반응 데이터를 활용하여 주목할 만한 식품 후보를 조기에 탐색하고, 데이터 기반으로 트렌드 가능성을 판단하는 것을 목표로 했습니다.

대시보드에서는 오늘의 인기상품, 예비 인기권 상승 후보, 관심 증가 상품, 가격 확인 필요 상품을 확인할 수 있습니다. 
사용자는 상품별 순위 변화, 검색 반응, 쇼핑 반응, 가격 데이터 연결 여부를 비교해 진입 타이밍을 판단할 수 있습니다.

## 데이터 수집 방식

EarlyPick은 별도의 Flask 서버를 구축하지 않고, Python 기반 데이터 수집 스크립트를 통해 외부 API를 직접 호출하는 방식으로 데이터를 수집했습니다.

Python 수집 스크립트
→ 외부 API 호출
→ JSON 응답 수신
→ 필요한 컬럼 정제
→ CSV 저장
→ 분석 및 대시보드 데이터로 활용

수집한 데이터는 다음과 같습니다.

| 구분               | 수집 방식                      | 활용 목적                   
| ------------------ | -------------------------------| ---------------------------------
| 네이버 Top20        | CSV 수집·정제                  | 이미 인기권에 진입한 상품 확인       
| 네이버 21~200위     | CSV 수집·정제                  | Top20 진입 전 예비 인기권 후보 관찰 
| 네이버 검색 반응     | Naver DataLab Search API      | 상품별 검색 관심 흐름 확인         
| 네이버 쇼핑 반응     | Naver DataLab Shopping API    | 상품별 쇼핑 관심 흐름 확인         
| KAMIS 가격 데이터    | KAMIS API                     | 도매·소매 가격 흐름 확인          
| 도매시장 경락가      | 공공데이터포털 API              | 원물 가격 및 시장 흐름 확인        
| SNS 후보 신호        | 수동 수집·정제                 | 초기 관심 후보 보조 신호로 활용      

## 데이터 분석 구조
Top20 데이터는 이미 인기권에 진입한 상품을 확인하는 기준 데이터로 활용했고, 21~ 200위 데이터는 Top20 진입 전 상승 후보를 관찰하기
위한 데이터로 활용했습니다 
수집된 상품명은 그대로 분석하기 어려워 기준 품목명으로 표준화하고, 상품 유형을 원물·가공식품·수산물·축산물·브랜드 상품·건강기능식품
등으로 분류했습니다.

Top20 데이터
→ 인기권 진입 상품 확인

21~200위 데이터
→ 예비 인기권 상승 후보 관찰

검색·쇼핑 반응
→ 실제 관심 흐름 확인

가격 데이터
→ 원물 품목의 시장 흐름 확인

## 분석 기법
EarlyPick은 다중 데이터 비교 분석 방식으로 설계했습니다. 

상품별로 순위 구간, 반복 등장 여부, 검색·쇼핑 반응, 가격 데이터 연결 여부를 비교해 우선 관찰할 트렌드 후보를 선별했습니다.
분석 기준
- 상품명 표준화
- 상품 유형 분류
- 순위 구간 분석
- 반복 등장 여부 분석
- 검색·쇼핑 반응 비교
- 가격 데이터 연결 여부 판단
- 규칙 기반 판단 지표 생성

현재 MVP에서는 예측보다 규칙 기반 판단 지표를 중심으로 상품 상태를 분류했습니다.
판단 지표
- 관심 증가
- 우선 관찰
- 관찰 필요
- 가격 확인 필요
- 판단 보류
- 분석 제외

## 분석 결과
EarlyPick은 2026.04.23~2026.06.09 기간 동안 네이버 21~ 200위 예비 인기권 데이터를 중심으로 8,640건의 상품 스냅샷을 수집했습니다.

분석 결과 , 현재 순위 구간은 이후 상승 가능성과 연관이 있음을 확인했습니다. 특히 21~ 50위 구간 상품은 이후 50위권에 재 진입하거나 유지되는 비율이 높아, 우선 관찰 후보로 가능하다고 판단했습니다. 

또한 Top20 데이터는 이미 인기권에 진입한 상품을 확인하는 기준 데이터로 활용하고, 21~ 200위 데이터는 Top20 진입 전 후보를 조기에 관찰하는 데이터로 활용했습니다.

이를 통해 EarlyPick은 단순 인기상품 확인 서비스가 아니라, 예비 인기권 데이터를 기반으로 우선 관찰해야 할 식품 트렌드 후보를 선별하는 서비스로 방향을 구체화했습니다.

## 서비스 이용 흐름 
대시보드 접속
→ 오늘의 인기상품 및 예비 인기권 후보 확인
→ 관심 상품 선택
→ 상세보기에서 순위·검색·쇼핑·가격 데이터 확인
→ EarlyPick 판단 지표 확인
→ 관심 상품 저장
→ 관심 목록에서 변화 추적
→ 상품화·가격 확인·광고 검토 여부 판단

사용자는 대시보드에서 오늘의 인기상품과 예비 인기권 상승 후보를 확인합니다. 
관심 있는 상품을 선택하면 상세보기 화면에서 순위 변화, 검색·쇼핑 반응, 상품 유형, 가격 데이터 연결 상태를 확인할 수 있습니다.
저장한 상품은 관심 목록에서 따로 관리할 수 있으며, 이후 상품화 가능성, 원재료 확인, 광고 집행 여부를 판단하는 데 활용할 수 있습니다.

## 회고

초기에는 SNS 신호와 시장가 데이터를 함께 활용하는 구조를 검토했지만, SNS 데이터의 수집량과 신뢰도, 가격 데이터와의 직접 연결 가능성에 한계가 있음을 확인했습니다.

이에 따라 현재 MVP는 네이버 데이터랩 기반의 데이터 수집·정제·비교 분석·규칙 기반 판단을 중심으로 방향을 조정했습니다.

이번 프로젝트를 통해 단순히 데이터를 많이 수집하는 것보다, 분석 가능한 기준으로 정제하고 서비스 판단에 활용할 수 있는 지표를 설계하는 과정이 중요하다는 점을 배웠습니다.

## 향후 계획

현재는 규칙 기반 분석 MVP 단계이며, 향후 데이터가 충분히 누적되면 머신러닝 기반 상승 후보 예측과 AI 인사이트 자동 생성 기능으로 확장할 예정입니다.
